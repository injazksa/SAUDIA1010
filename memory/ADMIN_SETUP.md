## ✅ الخطوات الأولى (مرة واحدة فقط)

### 1️⃣ رفع الموقع على Netlify عبر Git
- اربط مستودع GitHub/GitLab بـ Netlify (إذا لم يكن موصولاً).
- بمجرد الرفع، انتقل للوحة Netlify → موقعك.

### 2️⃣ تفعيل Netlify Identity
1. من لوحة Netlify → موقعك → **Integrations** (أو Site settings)
2. ابحث عن **Identity** واضغط **Enable Identity**
3. تحت **Registration preferences** → اختر **Invite only** ← مهم جداً
4. تحت **External providers** (اختياري): يمكنك تفعيل Google/GitHub للدخول السريع

### 3️⃣ تفعيل Git Gateway
1. في نفس صفحة Identity → انزل لـ **Services**
2. اضغط **Enable Git Gateway**
3. هذا يسمح للوحة التحكم بحفظ التغييرات في Git تلقائياً

### 4️⃣ دعوة إيميلك
1. في Identity → اضغط **Invite users**
2. أدخل البريد: `Visa@saudia-visa.com`
3. سيصلك إيميل تفعيل — افتحه، اضبط كلمة المرور التي تريدها، وتم ✅

## 🔒 الأمان

- ✅ **رابط سرّي**: `/admin-sv22128/` (لا يظهر في Google)
- ✅ **Invite-only**: لا أحد يستطيع التسجيل إلا من تدعوه أنت
- ✅ **robots.txt**: يمنع فهرسة اللوحة
- ✅ **Noindex headers**: حتى لو وجد أحد الرابط، Google لن يفهرسه
- ✅ **Git trail**: كل تعديل محفوظ في Git (يمكن التراجع عنه)

### نصائح إضافية:
1. **فعّل 2FA** في حساب Netlify نفسه
2. **غيّر كلمة المرور** كل 3 أشهر
3. **لا تدخل اللوحة من WiFi عام** بدون VPN

## 🆘 حل المشاكل

### "Login failed"
- تأكد من تفعيل Git Gateway في Netlify
- تأكد من تفعيل Identity في وضع Invite Only

### "Config file not found"
- تأكد أن الموقع المنشور يحتوي على `/admin-sv22128/config.yml`
- تأكد أن Netlify بَنَى بنجاح (تحقق من لوحة Deploys)

### "التعديلات لا تظهر"
- التعديلات تحتاج ~30 ثانية ليُعاد البناء ويُنشر الموقع الجديد
- جرّب Ctrl+F5 لإجبار المتصفح على تحديث الكاش
